MySQL - Error 1045 (28000)

Posted on 2017-01-20 Edited on 2019-10-13

What happened?

Alright, I don’t know what happened when trying to login MySQL. And it shows that message: MySQL - Error 1045 (28000) using password: YES. I am pretty sure that user/pwd pair is correct. After searching and studying some articles and forum discussions. I found it might cause by broken SQL or reinstall SQL. The interesting is I can’t remember I had done that before.

How to solve?

Here some steps show how to restore your SQL.

After stopping your SQL daemon, then we need to start SQL without grant table.

After that, login your SQL as root.

We need to reset root pwd now.

Let root regain the access right for your databases.

Update the permission & exit SQL

Re-login to your SQL

Still doesn’t work!

I believe you can use your root pwd to login SQL now, but if you still can’t login, keep going through following steps. Sorry for your back luck. You need to check /etc/mysql/debian.cnf file in your system and find [client] section. Use this account to login.

Input debian-sys-maint pwd. and repeat these steps. We need to reset root pwd now.

Let root regain the access right for all of your databases.

Update the permission & exit SQL

Re-login to your SQL
